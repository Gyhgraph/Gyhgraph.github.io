博客开张

在空闲时总想着折腾一个个人博客来写写一些有的没得，前后折腾了三次总算是开起来了，特此写一篇博客来纪念一下哈哈。 只有在实际用的时候学到的东西，才能称得上自己的东西。碰到从来都没有碰过的编程语言，凭着代码的规律和试错，可以小修小补，然后达到想要的效果，这过程实在是太痛苦了，一点一点的文档Wiki去翻，在提交代码的时候发现自己只会 clone 下来，不会 push，无奈只能再去看 git 文档，有子文件夹、子仓库模块，然后就又卡在哪，上一次搁置就是因为提交代码时把修改了两三天的 Hugo 主题源码给弄丢了，碰上马上要开学，没有心情再重新构建一遍就一直鸽到了现在。 这开博客的过程对于一个没有学系统过编程的人来说简直就是一场噩梦，跟小学时有背书课一样艰难。 我为什么要建一个博客？可能最开始就是觉得博客这个东东很酷，可是慢慢改变了我的想法，博客是一个载体，上面的文字，作者写出来想要表达的思想才是我需要的，博客界面简洁，有需要的基本功能就足够，那这样的话其实并不是很难，先使用一个主题，对于自己构建来说大概要到后面学过 CSS 语言后才会试着去做。 写着写着发现自己是真没啥逻辑，想到哪文字跟着蹦到哪。不怕。谁叫我有了一个博客了呢，表达不出来，不仅是自己脑袋里没有，也不是自己没有表达过不敢去表达出来害怕出错闹出词不达意的笑话（重新表述一下吧。除了一些很有天赋的人来说，对于大部分人在一些偏正式不正式的场合，需要去提前准备，用在平时“积累”，这种积累感觉是平时多看、多思考，然后在用时你才有话可说并且很有水到渠来的那种感觉，然后在此基础上你与陌生人的对话才会像好友闲谈说话那样自然，鼓起勇气发现自己有话题可说，有观点能表达出来，才不会让下次更不敢去表达）。扯着扯着像扯到英语口语问题上了。笑 这大概就是我要写博客的原因了（ ： 后面大概会去听一些逻辑方面的课程… 博客到底要写哪方面的东西呢？ 姑且就先列一个不定期修改的博客开张计划： 抽出时间多看些书，然后把想法、学到的东西记录到博客上。 加一个英文分类，来把博客的内容手工翻译为英文版的。 …
